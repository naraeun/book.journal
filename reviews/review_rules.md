# 리뷰 작성 규칙

## 파일 구조

### 책
- 파일 경로: `reviews/{연도}/{번호}.md`
- 헤더: `# {제목} — {작가}`
- 작가명에 역할 표기((지은이), (옮긴이) 등)는 제외
- 메타데이터: 번호, 날짜, 카테고리, 블로그 링크를 bold 목록으로 작성
- 메타데이터 아래 `---` 구분선 후 본문 시작

### KBS 라디오 극장
- 파일 경로: `reviews/drama/radio_theater/{제목}.md`
- 헤더: `# {제목} — KBS 라디오 극장`
- 메타데이터: 날짜, 방송, 원작(book 리뷰 링크), 블로그 링크, 출연진을 bold 목록으로 작성
- 출연진은 `역할: 성우명` 형식으로 들여쓰기 목록
- 메타데이터 아래 `---` 구분선 후 본문 시작

### 드라마
- 파일 경로: `reviews/drama/drama/{제목}.md`
- 헤더: `# {제목}`
- 메타데이터: 날짜, 플랫폼, 방영연도, 연출, 작가, 원작(book 리뷰 링크), 블로그 링크를 bold 목록으로 작성
- 메타데이터 아래 `---` 구분선 후 본문 시작

### EBS 위대한 수업
- 파일 경로: `reviews/greatminds/{제목}.md`
- 헤더: `# {제목} — EBS 위대한 수업`
- 메타데이터: 날짜, 강연자, 방영연도, 시청연도, 블로그 링크를 bold 목록으로 작성
- 구성: 메타데이터 안에 `**구성**` 항목으로 각 회차 주제를 들여쓰기 목록으로 작성
- 메타데이터 아래 `---` 구분선 후 본문 시작

### 웹툰
- 파일 경로: `reviews/webtoon/{제목}.md`
- 헤더: `# {제목}`
- 메타데이터: 날짜, 작가, 플랫폼, 작품연도, 읽은연도, 블로그 링크를 bold 목록으로 작성
- 메타데이터 아래 `---` 구분선 후 본문 시작

## 날짜 형식
- 모든 날짜는 ISO 8601 형식 사용: `YYYY-MM-DD` (예: `2026-04-22`)
- 월·일은 반드시 두 자리 제로패딩: `3월` → `03`, `5일` → `05`
- `YYYY.M.D` 형식 사용 금지

## 본문
- 단락 사이는 빈 줄 하나로 구분 (HTML `<p>` — 간격 벌어짐)
- 같은 단락 안에서 줄바꿈만 하려면 줄 끝에 공백 두 칸 (HTML `<br>` — 붙어있되 줄만 바뀜)
- 빈 줄 두 줄 이상 사용하지 않음
- 보이지 않는 유니코드 공백 문자(zero-width space 등) 제거

## 인용 구절 (`>`)
- 각 인용은 독립 블록으로 작성
- 인용 사이는 빈 줄 하나로 통일
- 같은 맥락의 연속 인용도 별도 `>` 블록으로 분리
- 하나의 인용 안에서 줄바꿈이 필요하면 `>` 빈 줄(`>` + 빈 줄)로 단락 구분

## 목록
- 나열 항목은 `-` 순서 없는 목록 사용
- `-` 뒤에 반드시 공백 한 칸을 넣어야 마크다운 목록으로 인식됨 (`-항목` ✗ → `- 항목` ✓)
- 책 목록: `- {작가} - {제목}` 형식

## 관련 도서 링크
- 본문 흐름에 맞게 두 가지 방식 사용 가능
  - 인라인 링크: `[제목](상대경로)` — 문장 속에서 자연스럽게 언급할 때
  - 목록 링크: `- [제목 - 작가](상대경로)` — 여러 권을 나열할 때
- 상대경로 기준
  - 같은 연도: `[제목](1363.md)`
  - 다른 연도: `[제목](../2022/2158.md)`
- 리뷰가 없는 책은 링크 없이 텍스트로만 표기

## 추가 노트
- 본문 뒤에 책에서 얻은 배경지식, 연표, 참고 정보 등을 덧붙일 때 사용
- 본문 마지막 단락 뒤 `---` 구분선으로 노트 영역 시작
- 각 주제별로 `###` 소제목을 붙이고, 주제 사이는 `---` 구분선으로 구분
- `========`, `=====` 등 비표준 구분선 사용 금지

## 재독 업데이트
- 책을 다시 읽고 내용을 추가할 때 사용
- 메타데이터의 `**날짜**` 바로 아래에 `**업데이트 날짜**` 필드를 추가하고 최신 업데이트 날짜를 기입
- 본문 마지막 단락 뒤 `---` 구분선으로 재독 영역 시작
- `### YYYY-MM-DD` 소제목으로 시작 (날짜만 표기)
- 이후 내용은 기존 본문 규칙(단락, 목록, 인용 등)을 동일하게 따름
- 여러 번 재독한 경우 날짜별로 `###` 소제목을 추가하고, 사이는 `---` 구분선으로 구분

## 외부 미디어 링크
- 본문에서 언급하는 영상, 음악 등 외부 콘텐츠는 목록 링크로 작성
  - YouTube: `- [제목](https://youtu.be/...)`
  - Apple Music: `- [Apple Music: 제목](https://music.apple.com/...)`
- 본문 흐름 속에 자연스럽게 배치 (별도 섹션으로 분리하지 않음)
- 링크 텍스트에 플랫폼명 또는 콘텐츠 유형을 포함해 구분
- bare link (마크다운 링크 없이 URL만 붙여넣기) 사용 금지
